# 使用方法

## 記事の作成

### 基本的な記事の作成

1. `articles/` ディレクトリに新しい `.md` ファイルを作成します
2. ファイル名は、URLのスラッグとして使用されます（例: `my-article.md` → `/articles/my-article`）

### フロントマターの設定

各記事の先頭に、YAMLフロントマターを記述します：

```markdown
---
title: 記事のタイトル
date: 2025-11-16
tags: [Next.js, React, Markdown]
---

# 見出し

ここから記事の本文を書きます。
```

#### フロントマターのフィールド

- **title** (必須): 記事のタイトル
- **date** (必須): 公開日（YYYY-MM-DD形式）
- **tags** (オプション): タグの配列

### Markdownの記法

KumoCodeは標準的なMarkdown記法と、いくつかの拡張機能をサポートしています。

#### 見出し

```markdown
# h1見出し
## h2見出し
### h3見出し
#### h4見出し
##### h5見出し
###### h6見出し
```

#### 強調

```markdown
**太字**
*斜体*
~~取り消し線~~
```

#### リスト

```markdown
# 順序なしリスト
- アイテム1
- アイテム2
  - ネストしたアイテム

# 順序付きリスト
1. 最初のアイテム
2. 次のアイテム
3. 三番目のアイテム
```

#### リンクと画像

```markdown
[リンクテキスト](https://example.com)
![画像の説明](path/to/image.jpg)
```

#### コードブロック

````markdown
```javascript
function hello() {
  console.log("Hello, KumoCode!");
}
```
````

サポートされている言語：
- JavaScript / TypeScript
- Python
- Java
- C / C++ / C#
- Go
- Rust
- Ruby
- PHP
- その他多数（Highlight.jsがサポートする全ての言語）

#### インラインコード

```markdown
`inline code` のように記述します。
```

#### 引用

```markdown
> これは引用です。
> 複数行にわたることもできます。
```

#### テーブル

```markdown
| ヘッダー1 | ヘッダー2 | ヘッダー3 |
|----------|----------|----------|
| セル1    | セル2    | セル3    |
| セル4    | セル5    | セル6    |
```

#### 水平線

```markdown
---
```

### 数式の記述

KumoCodeはKaTeXを使用して、LaTeX形式の数式をサポートしています。

#### インライン数式

```markdown
質量とエネルギーの等価性: $E = mc^2$
```

#### ブロック数式

```markdown
$$
\int_{-\infty}^{\infty} e^{-x^2} dx = \sqrt{\pi}
$$
```

## テーマの変更

### ユーザーインターフェースからの変更

1. ページ右上のテーマ切り替えボタン（☀️/🌙アイコン）をクリック
2. ライトモードとダークモードが切り替わります

### デフォルトテーマの設定

`.env`ファイルでデフォルトテーマを設定できます：

```env
NEXT_PUBLIC_DEFAULT_THEME=ocean
```

利用可能なテーマ：
- **ocean**: 青を基調とした海のテーマ
- **default**: シンプルなデフォルトテーマ
- **sunset**: オレンジを基調とした夕焼けテーマ
- **cyberpunk**: ネオンカラーのサイバーパンクテーマ

## 開発のヒント

### ホットリロード

開発サーバー（`npm run dev`）を使用している場合、ファイルを保存すると自動的にブラウザがリロードされます。

### 記事のプレビュー

1. 開発サーバーを起動します
2. ブラウザで http://localhost:3000 にアクセス
3. 記事一覧から確認したい記事をクリック

### デバッグ

ブラウザの開発者ツール（F12キー）を使用して、エラーやコンソールログを確認できます。

## ベストプラクティス

### ファイル名の命名規則

- 小文字を使用
- 単語の区切りはハイフン（-）を使用
- 例: `getting-started.md`, `api-reference.md`

### 画像の配置

画像は `public/images/` ディレクトリに配置し、以下のように参照します：

```markdown
![説明文](/images/screenshot.png)
```

### タグの使用

記事を分類しやすくするため、適切なタグを付けることを推奨します：

```yaml
tags: [Next.js, React, Tutorial]
```
