# Claude Code エージェント設定

このディレクトリには、Claude Codeで使用するカスタムエージェントとコマンドが含まれています。

## 📁 ディレクトリ構成

```
.claude/
├── agents/              # カスタムエージェント定義
│   ├── code-reviewer.md       # 汎用コードレビューエージェント
│   ├── reviewer-kozaki.md     # 🦊 狐崎煌羽（プロダクト責任者）
│   ├── reviewer-yukimura.md   # 🎀 雪村綾乃（UX/UIデザイナー）
│   └── reviewer-onizuka.md    # 🌿 鬼塚蓮司（やさしい関西弁エンジニア）
├── commands/            # スラッシュコマンド
│   └── (今後追加予定)
└── README.md           # このファイル
```

## 🤖 利用可能なエージェント

### 1. 汎用コードレビューエージェント
**ファイル**: `agents/code-reviewer.md`

包括的なコードレビューを実施します。

**レビュー観点**:
- コード品質（可読性、保守性、一貫性）
- アーキテクチャと設計（SOLID原則、デザインパターン）
- パフォーマンス（効率性、メモリ使用、スケーラビリティ）
- セキュリティ（脆弱性、認証・認可、データ保護）
- テスト（カバレッジ、テストの質、テスタビリティ）
- ドキュメント（README、コメント、API仕様）

### 2. 🦊 狐崎煌羽 - プロダクト責任者
**ファイル**: `agents/reviewer-kozaki.md`

ビジネスインパクトを重視した厳格なレビューを実施します。

**特徴**:
- ビジネス価値とROIを評価
- リスク評価とリソース配分の妥当性
- KPI/KGIへの貢献度を分析
- 実行判断（Go/Hold/Rework）を明確化

**適した場面**:
- 新機能のビジネス価値を評価したい時
- リリース前の最終判断が必要な時
- プロダクトの方向性を検証したい時

### 3. 🎀 雪村綾乃 - UX/UIデザイナー
**ファイル**: `agents/reviewer-yukimura.md`

使いやすさと映えを重視した明るいレビューを実施します。

**特徴**:
- ビジュアルの一貫性とブランド感
- UIフローの直感性と初見ユーザー体験
- アクセシビリティとマイクロコピー
- 感情設計と満足度

**適した場面**:
- UIコンポーネントの使いやすさを確認したい時
- デザインの一貫性をチェックしたい時
- ユーザー体験を向上させたい時

### 4. 🌿 鬼塚蓮司 - やさしい関西弁エンジニア
**ファイル**: `agents/reviewer-onizuka.md`

技術的な実現性を重視した安心感のあるレビューを実施します。

**特徴**:
- 実装の正しさと安定性
- パフォーマンスとスケーラビリティ
- コード構造とテストの妥当性
- 前向きな改善策の提案

**適した場面**:
- 技術的な実装の妥当性を確認したい時
- パフォーマンスが心配な時
- チームメンバーを励ましながらレビューしたい時

## 🚀 使い方

### エージェントの実行方法

Claude Codeのチャットで、以下のように指示してください：

```
# 汎用レビューエージェントを使用
code-reviewer エージェントを使ってこのPRをレビューしてください

# 狐崎煌羽（プロダクト責任者）を使用
reviewer-kozaki エージェントでビジネスインパクトを評価してください

# 雪村綾乃（UX/UIデザイナー）を使用
reviewer-yukimura エージェントでUIコンポーネントをチェックしてください

# 鬼塚蓮司（やさしい関西弁エンジニア）を使用
reviewer-onizuka エージェントで技術的にレビューしてください
```

### レビュー対象の指定

```
# ファイル指定
src/components/Button.tsx をレビューしてください

# ディレクトリ指定
src/features/payment/ 配下をレビューしてください

# PR全体
現在のPRをレビューしてください

# 特定のコミット
コミット abc123 をレビューしてください
```

### フォーカスエリアの指定

```
# セキュリティ重視
セキュリティの観点でレビューしてください

# パフォーマンス重視
パフォーマンスを中心にレビューしてください

# 可読性重視
コードの可読性を重点的にチェックしてください
```

## 📝 レビューの活用方法

### 1. 多角的なレビュー
異なるペルソナのエージェントを組み合わせて使うことで、多角的な視点からレビューを受けられます。

```
1. まず reviewer-onizuka で技術的な正しさを確認
2. 次に reviewer-yukimura でUIの使いやすさをチェック
3. 最後に reviewer-kozaki でビジネス価値を評価
```

### 2. ステージ別の活用
開発フェーズに応じて適切なエージェントを選択：

- **開発初期**: code-reviewer（設計とアーキテクチャ）
- **実装中**: reviewer-onizuka（技術的な実装）
- **UI実装**: reviewer-yukimura（デザインと使いやすさ）
- **リリース前**: reviewer-kozaki（ビジネスインパクト）

### 3. チーム学習
レビュー結果をチーム内で共有し、共通の知見として活用：

- レビュー指摘事項のパターン分析
- ベストプラクティスの蓄積
- チーム内のコーディング規約への反映

## ⚙️ カスタマイズ

### エージェントの追加
新しいエージェントを追加する場合：

1. `.claude/agents/` に新しい `.md` ファイルを作成
2. ペルソナ、レビュー観点、出力フォーマットを定義
3. このREADMEに使い方を追記

### エージェントの編集
既存のエージェントをカスタマイズする場合：

1. 対応する `.md` ファイルを編集
2. プロジェクト固有のルールやガイドラインを追加
3. 評価基準やフォーカスエリアを調整
